# 领域全景

> **任务类型**：领域全景
> **触发条件**：用户问"领域发展/趋势/生态""XX 方向有哪些代表性工作"

---

## 执行深度

Round 1(仓库+综述+最新进展) → Final

## 子 agent 策略

- **按来源拆分，不按对象分组**：
  - A（经典综述与 taxonomy）—— 该领域的经典分类、里程碑综述、代表性论文
  - B（开源仓库与生态）—— awesome lists、高星开源项目、工具链
  - C（近 2-3 年最新进展）—— 顶会最新论文、新方向、争议焦点
- **可选**：D（遗漏方向与反例）—— 当用户问"是否有遗漏"时增设

## 分析框架

1. **领域定义与边界**：这个领域研究什么、不研究什么
2. **Taxonomy / 分类体系**：主流分类方式是什么、各分支的代表性工作
3. **技术主线**：从早期到现在的 2-3 条主要技术演进路线
4. **当前状态**：领域处于什么阶段（萌芽/快速发展/成熟/瓶颈）
5. **开放问题与研究机会**：尚未解决的关键问题、潜在研究方向

## Pre-final Gate

- Taxonomy 已建立，每个分支有代表性工作
- 技术主线已梳理，有时间线感
- 开放问题已列出，标注了当前研究覆盖程度

## 输出重点

压缩为"领域定义—taxonomy—技术主线—当前状态—开放问题"。按 taxonomy 组织，不按单篇论文组织。不要求跨领域映射表。

## 典型编排

```
Round 1:
  A：经典综述和 taxonomy
  B：开源仓库和生态
  C：最新论文和趋势

Checkpoint:
  分类完整 → Final
  有遗漏方向 → 补充 D

Final:
  领域全景笔记（按 taxonomy 组织）
```
