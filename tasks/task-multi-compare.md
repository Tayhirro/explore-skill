# 多对象对比分析

> **任务类型**：多对象对比分析
> **触发条件**：用户给出 ≥2 个对象并要求逐一分析或对比

---

## 执行深度

Round 1(A+B+C) → Round 2(逐对象逐核心单元) → Verification → Final

这是最重的任务类型，包含完整的 Verification 轮。

## 子 agent 策略

- **必须**：
  - A（结构理解）—— 贯穿全部对象的主线概括，或按对象分组（A1/A2/...）
  - B（对比与定位）—— 各对象之间的关系、共同基线、差异点
  - C（理论与跨学科依据）—— 各对象核心机制的理论支撑对比
- **可选**：
  - D/E（按对象分组的辅助角色）—— 当对象数量 ≥4 时增设

## 核心单元抽取

每个对象独立抽取核心单元并分级。Round 2 中每个对象的每个核心单元都需要独立的分析段落。

## Pre-final Gate（比其他类型更严格）

- Verification 轮已执行：至少 5 条关键断言的引用核查，每条标记 SUPPORTED / PARTIAL / UNSUPPORTED
- 每个对象的核心单元都有独立的"痛点→修改点→机制→证据→边界"分析链
- 跨领域概念映射表已填充（≥2 领域）
- 执行清单全部打勾

## 输出重点

按对象组织，每个对象一个分析单元。附加跨对象对比表（核心机制、适用场景、优劣、理论依据）。附加变化/贡献总结。

## 典型编排

```
Round 1:
  A：全部对象结构理解（或 A1/A2 分组）
  B：跨对象 baseline 对比
  C：理论依据对比

Checkpoint:
  每个对象抽取核心单元 → 分级 → promoted

Round 2:
  D：对象 1 的核心单元逐一分析
  E：对象 2 的核心单元逐一分析
  F：对象 3 的核心单元逐一分析（如有）

Verification:
  5-10 条关键断言核查

Final:
  综合笔记 + 对比表 + 映射表
```
