# 单对象概览

> **任务类型**：单对象概览
> **触发条件**：用户问"这篇论文/系统/技术讲什么""大概在做什么"

---

## 执行深度

Round 1(A) → Final

这是最轻量的任务类型。目标是快速建立对对象的整体认知，不需要深度验证或对比。

## 子 agent 策略

- **必须**：A（结构理解）—— 概括对象主线、章节/模块结构、核心断言、输入输出
- **不需要**：B（对比）、C（跨学科理论）
- **判断**：如果材料短（<10 页）且结构简单，主 agent 可以直接完成 A 的工作而不派发子 agent

## Pre-final Gate

- A 的输出已覆盖对象的主线和核心断言
- 最终笔记能回答"它在做什么、怎么做、主要贡献是什么"

## 输出重点

压缩为"对象—主线—结构—贡献"四要素。不展开外部对照、不展开创新点逐一分析、不要求跨领域映射表。

## 典型编排

```
Round 1:
  A：原文/对象结构理解 → 主线、章节、claim、贡献

Checkpoint:
  A 足够 → resolved → Final

Final:
  输出对象概览笔记
```
